# 聊天 UI 约束

## 目标

为后续聊天类界面生成、重绘、对齐提供统一约束，避免出现“像微信但不是微信官方”的失真设计。

## 平台基线

- 默认按 iPhone 16 尺寸与视觉比例进行对齐
- 默认按最新 iOS 风格处理顶部状态栏、圆角、留白与系统信息区
- 顶部边框必须包含时间、电量、Wi-Fi、蜂窝信号等系统状态信息

## 私信规则

- 若对话中没有明确的“自己”身份信息，默认不生成回复内容
- 只有当“自己”的身份、昵称、头像或说话角色被明确给出时，才允许生成回复气泡
- 回复场景中，第一人称头像允许自定义
- 若未指定第一人称头像，保留头像占位规则，但不要擅自虚构真人信息

## 微信风格约束

- 微信相关界面优先参考官方公开截图，不要使用非官方二创按钮样式
- 点赞、评论、消息列表、群聊、语音输入等控件应尽量贴近微信官方公开版本
- 不允许把通用社交 UI 误当作微信 UI

## X / 推特私信约束

- X 相关界面优先参考官方公开截图和公开产品风格
- 当前公开素材库可作为 `X` 的品牌风格、首页信息流、社区发现的基线参考
- 若要实现 `X 私信`，在没有更直接官方素材前，不要伪造“官方私信截图”
- 如果只能做近似实现，必须在提案或 sync 阶段明确标注“基于官方公开风格的近似私信实现”

## 小红书私信约束

- 小红书相关界面优先参考官方公开截图
- 已有消息列表官方截图时，优先贴近其顶部结构、列表密度、Tab 组织与底部导航
- 若进入私信详情页而缺少直接官方截图，允许在消息列表风格基础上做延展，但必须保持品牌配色、间距和组件气质一致
- 不允许把通用 IM 风格直接替代小红书私信风格

## 文档与代码同步要求

- 如果界面实现与素材库参考不一致，需要在提案或 sync 阶段明确记录差异原因
- 如果采用了非官方近似样式，必须在文档中标注“近似实现”，不能伪装成官方还原

## 推荐流程

1. 先从素材库选择最接近的官方参考图
2. 在 `/explore` 或 `/propose` 阶段记录目标界面和差异点
3. 在 `/apply` 阶段落实 UI
4. 在 `/sync` 阶段检查控件语义、布局和视觉是否与素材一致
